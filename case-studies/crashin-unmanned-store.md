# 케이스 스터디 — 무인 매장 통합 시스템 (크래쉬인)

> 예약 · 결제 · 출입(도어락) · 키오스크를 하나로 묶은 **풀스택 무인 매장 운영 시스템**

## 도전

직원 없는 무인 매장을 운영하려면 **예약 → 결제 → 출입 → 정산**이 사람 개입 없이 자동으로 맞물려야 합니다. 웹·키오스크·도어락·결제가 하나의 시스템으로 동작해야 하고, **네트워크가 끊겨도 고객 입장은 막히면 안 됩니다.** 또한 여러 지점을 한 곳에서 관리할 수 있어야 합니다.

## 해결

웹 · 키오스크 · IoT 도어락 · 결제를 하나로 통합한 멀티 컴포넌트 시스템으로 구축했습니다.

### 웹 예약
- 좌석 선택 · **선점(hold)** · 이용권 구매 · QR 발급 · PG 결제

### 키오스크 (전용 단말)
- **키오스크 모드 잠금** — 단말을 전용 모드로 고정, 부팅 시 자동 실행
- **오프라인 내성** — 로컬 DB로 네트워크 단절 시에도 예약 조회·입장 처리, 복구 시 백엔드와 자동 동기화
- 실시간 동기화 — 서버 상태 변화를 키오스크에 즉시 반영

### 출입 제어 (도어 키패드 + 전자석 도어락)
- 결제·인증이 끝나면 **출입 단말(도어 키패드)** 과 **전자석 도어락(EM lock)** 을 제어해 자동 개폐
- **권한을 현장 단말에 미리 동기화** — 인터넷이 끊겨도 단말이 로컬 권한으로 즉시 문을 엶 (온라인 복구 시 재동기화). "클라우드가 권한을 정하고, 현장 장비가 그걸 들고 독립적으로 작동"
- QR · 코드 기반 출입 · 입장/퇴장 로그 기록

### 다지점 관리자
- 지점(Branch) · 예약 · 결제 · 이용권 · 좌석 · 정산을 한 대시보드에서 통합 관리

## 결과
- **직원 없는 24시간 무인 운영**
- 웹 · 키오스크 · 도어락 · 결제 · 관리자가 **하나의 시스템**으로 통합
- **오프라인 내성** — 네트워크 단절 상황에서도 고객 입장 지속
- 다지점 확장 가능한 구조

## 기술
Spring Boot · Android(Kotlin, 오프라인 로컬 DB · 키오스크 모드) · 전자석 도어락(EM lock) · 도어 키패드 단말(MQTT 권한 동기화) · 실시간 동기화(SSE) · PG 결제 · QR

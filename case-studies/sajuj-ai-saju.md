# 케이스 스터디 — AI 사주 · 명리 서비스 (사주J)

> 결정론적 명리 엔진 + LLM 해석을 결합한 **AI 사주 자체 서비스** (sajuj.com)

## 도전

AI로 사주·명리 서비스를 만들 때 두 요구가 충돌합니다.
- **정확성** — 명식(사주 구성)은 한 글자도 틀리면 안 되는 **결정론적 계산**
- **자연스러움** — 해석은 사람이 읽을 수 있는 **자연어 서사**

LLM에만 맡기면 명식 계산이 틀리고, 룰 엔진에만 맡기면 해석이 딱딱합니다. 여기에 실시간 상담·결제까지 더해야 합니다.

## 해결

**계산은 결정론 엔진, 해석은 LLM** 으로 역할을 분리했습니다.

- **결정론 명리 엔진** — 명식 · 궁합을 룰 기반 엔진으로 계산 (재현 가능 · 정확)
- **LLM 해석** — 계산된 명식을 LLM이 자연어로 풀이 (모델 토글 · 응답 캐싱으로 비용 최적화)
- **실시간 상담** — STOMP 채팅(종단간 암호화) + 진행 상태 실시간 표시
- **결제** — 이용권 · 정기 결제 (KCP)
- **멀티플랫폼** — 웹 · Android · iOS

## 결과
- **라이브 서비스 운영** (sajuj.com)
- 결정론(정확성) + LLM(자연스러움)의 **하이브리드 AI** 구조
- 기획 · 백엔드 · 프론트 · 앱 · AI · 결제를 아우른 **풀스택 자체 제품**

## 기술
Spring Boot · Next.js · LLM(Claude · OpenAI) · STOMP/SSE 실시간 · KCP 결제 · Android · iOS

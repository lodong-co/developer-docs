# 케이스 스터디 — 개발 지식 칼럼 플랫폼 (콕컷)

> 개발사가 직접 쓰는 IT·개발 칼럼 + SEO/GEO 자동화 **자체 서비스** (kokcut.com)

## 도전

개발사의 전문성과 신뢰를 콘텐츠로 보여주는 발행 플랫폼이 필요했습니다. 동시에 세 가지를 만족해야 했습니다.

- **신뢰** — 광고·낚시 없는 양질의 글만, 로동이 직접 발행
- **노출** — 검색엔진(구글·네이버)과 생성형 AI 검색 모두에 잘 잡히는 구조(GEO)
- **속도** — 이걸 0부터 만들지 않고, 재사용 모듈로 빠르게

## 해결

**콘텐츠 도메인을 재사용 모듈로 추출**하고, **발행 한 번이 SEO·색인·AI 노출까지 자동으로** 이어지게 설계했습니다.

- **신규 재사용 모듈** — 글/발행을 `ld-article-module`, SEO 자동화를 `ld-seo-module`로 추출(엔티티 비소유 제네릭) → 다음 서비스에서도 그대로 재사용
- **SEO 자동화** — 발행만 하면 메타 · canonical · 구조화 데이터(JSON-LD) · sitemap · RSS 자동 생성
- **색인 자동화** — IndexNow로 빙·네이버 발행 즉시 색인 · 구글 사이트맵 · 소유권 인증 메타 자동 삽입
- **AI 친화(GEO)** — AI 크롤러 허용 + `llms.txt`로 생성형 AI에 사이트 안내
- **신뢰 가능한 지표** — 봇·SSR 조회를 제외하고 사람 조회만 세는 비콘 + 중복 제거 정책
- **참여** — 댓글 · 대댓글 · 추천(업보트) · 소셜 로그인

## 결과

- 라이브 서비스 운영 (kokcut.com)
- 신규 재사용 모듈 2종(`ld-article` · `ld-seo`) 추출 → **모듈 생태계 확장**
- **글 한 번 발행 = SEO · 다중 검색엔진 색인 · AI 노출까지 자동**

## 기술

Next.js · Spring Boot · ld-article / ld-seo 모듈 · Redis 캐시 · IndexNow · JSON-LD · sitemap / RSS · 소셜 로그인

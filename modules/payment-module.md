# payment-module — 결제

국내 PG(KCP·토스·웰컴페이먼츠)를 **하나의 일관된 인터페이스로 통합**한 결제 모듈. PG마다 다른 기능 집합을 깔끔하게 추상화합니다.

> 실운영: **사주J**(KCP), **forme** 에서 라이브 가동 중.

## 주요 기능
- **다중 PG 통합** — KCP·토스·웰컴을 단일 클라이언트 인터페이스로. 설정값 하나로 PG 전환
- **선택적 기능의 우아한 처리** — 빌링(자동결제)·환불·부분취소처럼 PG마다 지원이 다른 기능을, 미지원 시 명확한 오류로 처리
- **인증서 수명주기 관리** — 인증서 만료 30일 전부터 사전 경고 로그. 만료/임박 상태를 운영자가 미리 인지
- **fail-fast 비밀 처리** — 인증서·키 경로가 없으면 기동을 중단하고 어떤 설정이 빠졌는지 명확히 안내. 키 자료를 절대 번들에 포함하지 않음
- **라이선스 기동 게이트** + **난독화 빌드** 지원

## 설계 노트
- **검증된 핵심 경로** — 결제 서명 생성/검증, 위변조 탐지, 결정성(determinism)을 테스트로 증명합니다.
- **운영 성숙도** — 외부 PG 호출의 안정성과 인증서 운영을 코드 차원에서 관리합니다.

## 도입
```gradle
implementation "com.lodong:payment-module"   // 저장소 접근·버전은 계약 시 안내
```
PG 자격증명은 호스트 앱이 보안 저장소에서 주입하며, 모듈은 인증서·키를 번들하지 않습니다.

> 본 문서는 기능 소개입니다. API 상세·예제·소스는 구독 계약 시 제공됩니다.

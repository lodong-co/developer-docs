# ld-article-module — 글 · 뉴스 콘텐츠

글 · 뉴스 · 칼럼의 **발행 라이프사이클을 표준화한 콘텐츠 모듈**. 작성 → 발행 → 노출까지의 공통 흐름과, **봇을 거른 신뢰 가능한 조회수**까지 갖춘 콘텐츠 인프라입니다.

> 실운영: **콕컷(kokcut.com)** 라이브 가동 중.

## 핵심 가치
- **빠른 구축** — 콘텐츠 서비스의 반복 기능(목록·검색·발행·조회수)을 즉시 장착
- **신뢰 가능한 지표** — 조회수가 봇·자동요청이 아닌 '사람'만 세어짐
- **SEO 친화** — `ld-seo-module`과 결합해 발행만 하면 검색·AI 노출 자동

## 기능
### 발행
- 초안(DRAFT) → 발행(PUBLISHED) → 보관(ARCHIVED) 상태 전이
- 발행 시점 고정 · 탑 노출(featured) · 카테고리 분류
- slug 자동 생성 + 중복 회피(유니크화)

### 조회 · 검색
- 발행글 목록(최신 · 핫스코어 랭킹) · 카테고리별 · 키워드 검색

### 조회수 정책 (사람만 집계)
- **봇·크롤러 제외**(User-Agent) + **중복 제거**(회원/방문자당 1회) 옵션
- 상세 조회(GET)가 아니라 **브라우저 비콘**으로 집계 → SSR·새로고침·크롤러 과다집계 방지
- 정책은 교체 가능한 확장점(SPI) — "매번 집계" / "1인 1회" 등 선택

## 설계 노트
- **엔티티 비소유** — 글 엔티티/저장은 호스트 앱이 소유, 모듈은 조회·발행검증·slug·조회수 정책만 제공(제네릭)
- 확장 지점 우선 — 조회수 정책 등 핵심 동작을 호스트가 재정의 가능
